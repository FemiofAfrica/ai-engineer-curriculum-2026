Module 8: Deployment & Basic MLOps

"Works on my machine" doesn't count. Containerise, deploy, monitor, and eval your AI app.

🎯 Before you start: This module assumes you've completed Modules 1–7. You should be comfortable with Python, terminal, git, API calls, and have built at least one AI project. This module takes that project and makes it deployable.
Code→ Dockerfile→ Build Image→ Push to Registry→ Deploy→ Monitor→ Eval & Improve

Core Concepts

Docker

Containerise your app with OS, Python, dependencies, and secrets. Reproducible from laptop to cloud.

Cloud Deploy

Railway, Render, Fly.io, Vercel. Pick one and learn its deploy flow, domains, scaling.

Secrets Management

API keys, DB URLs, model configs — never in code. Use env vars and platform secret stores.

Monitoring

Logs, error tracking, latency, cost per request. You can't improve what you don't measure.

Evaluation

Automated tests for your AI app: retrieval accuracy, hallucination rate, response quality, latency SLAs.

CI/CD

Automated deploy on push. Tests pass → build → deploy. No manual steps.

FastAPI AI App

main.py
from fastapi import FastAPI, HTTPException
from pydantic import BaseModel
from openai import OpenAI
import os

app = FastAPI(title="AI RAG API")
client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

class Query(BaseModel):
  question: str

class Answer(BaseModel):
  answer: str
  sources: list[str]
  tokens_used: int

@app.post("/ask", response_model=Answer)
async def ask(query: Query):
  # In production: retrieve from vector DB, construct prompt
  resp = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role":"user","content":query.question}]
  )
  return Answer(
    answer=resp.choices[0].message.content,
    sources=[],
    tokens_used=resp.usage.total_tokens
  )

Simple Eval Script

eval.py
# Checks if AI responses contain hallucinated or missing citations
import json, httpx

def evaluate_answer(question, answer, expected_sources):
  # 1. Check that every claim has a source citation
  has_citations = "[" in answer and "]" in answer

  # 2. Check that sources are real (not hallucinated)
  mentioned = set()
  import re
  for m in re.finditer(r'\[(.*?)\]', answer):
    mentioned.add(m.group(1).lower())
  real = set(s.lower() for s in expected_sources)
  hallucinations = mentioned - real

  return {
    "has_citations": has_citations,
    "hallucinated_sources": list(hallucinations),
    "missing_sources": list(real - mentioned),
    "passes": has_citations and len(hallucinations) == 0
  }

⚠️ Pitfalls

No .dockerignore. Your Docker context includes node_modules, .venv, .git — huge build times. Always have a .dockerignore.
Hardcoded secrets. API keys in code = security incident. Use env vars + platform secret manager.
No health checks. Your app can be running but broken (no DB connection, API key expired). Add /health endpoint and platform health checks.
Deploying without observability. No logs, no metrics, no error tracking. You're debugging blind. Add logging and at minimum a Sentry/Datadog integration.
No eval pipeline. You think it works because you tested 3 queries. Build an automated eval suite before you ship.

🏗️ How It All Connects

Your code travels from your computer to the cloud through the terminal and git.

1️⃣ What is the Terminal? Start Here

If you've never opened a terminal before, don't worry — you're in exactly the right place.

Your computer has two ways to talk to you:

  • GUI (Graphical User Interface) — windows, buttons, icons, the mouse. This is what you use every day: clicking folders, dragging files, opening apps.
  • Terminal (Command Line Interface) — a text-based way to talk to your computer. Instead of clicking, you type commands. The computer types back the result.

Think of the terminal as the direct channel to your computer's brain. The GUI hides complexity to make things visual. The terminal shows you everything — it's actually more powerful because you can say exactly what you want.

How to Open the Terminal on macOS

  1. Press ⌘ Cmd + Space to open Spotlight Search.
  2. Type "Terminal" and press Enter.
  3. A window pops up with black (or dark) background and white text. That's it! You're in the terminal.

You'll see something like macbook@username ~ % followed by a blinking cursor. That's called the prompt — it's waiting for your command.

💡 Tip: On Windows, you can use PowerShell or install Windows Subsystem for Linux (WSL). On Linux, the terminal is usually in your app menu as "Terminal" or accessible with Ctrl+Alt+T.

2️⃣ Your First Commands Do This

Let's move around your computer using only text.

Open your terminal and type these commands one at a time. Press Enter after each one.

bash — first-steps.sh
# "Print Working Directory" — shows where you are right now $ pwd /Users/macbook # "List" — shows files and folders in the current directory $ ls Applications Desktop Documents Downloads Music Pictures # "Change Directory" — move into a folder $ cd Desktop # Check you're in Desktop now $ pwd /Users/macbook/Desktop # Go back to your home folder $ cd .. # "Make Directory" — create a new folder $ mkdir ai-projects # Move into it $ cd ai-projects # "Touch" — create an empty file $ touch hello.py # List to see your new file $ ls hello.py
CommandWhat it does
pwdShow your current location (Print Working Directory)
lsList files and folders in the current directory
cd <name>Change directory — move into a folder
cd ..Go up one folder (back to parent)
mkdir <name>Create a new folder
touch <filename>Create an empty file
💡 Tip: You can press Tab to auto-complete file and folder names. Type cd De and press Tab — it finishes "Desktop" for you.

3️⃣ Moving Files Around Essential

Copy, move, rename, and delete files from the terminal.

bash — file-ops.sh
# Copy a file: cp source destination $ cp hello.py hello-backup.py # Move (or rename) a file: mv source destination $ mv hello.py main.py # Rename hello.py to main.py $ mv main.py ../ # Move main.py up one folder # Delete a file — ⚠️ WARNING: no undo! $ rm hello-backup.py
CommandWhat it does
cp <source> <dest>Copy a file from source to destination
mv <source> <dest>Move or rename a file
rm <file>Delete a file (⚠️ permanent — no trash bin)
rm -r <folder>Delete a folder and everything inside it
🚨 Critical: rm does not send files to the Trash. Once deleted, they're gone forever. There is no "undo" for rm. Always double-check before deleting.

4️⃣ Looking at Files Handy

View the contents of files without opening a text editor.

bash — view-files.sh
# "Concatenate" — print the whole file at once $ cat hello.py print("Hello, world!") # "Less" — scroll through a file (press q to quit) $ less README.md # "Head" — show first 5 lines $ head -5 data.csv # "Tail" — show last 5 lines $ tail -5 data.csv
CommandWhat it does
cat <file>Print the entire file to the terminal
less <file>Scroll through a file page by page (press q to exit)
head -N <file>Show the first N lines of a file
tail -N <file>Show the last N lines of a file

5️⃣ What is Version Control? Big Idea

Why every developer uses Git — and why you should too.

🎮

Like Save Points in a Game

You play a game, save at a checkpoint, try something risky, and if things go wrong — you load your save. Git does that for your code.

⏰

Time Machine for Your Code

Every commit is a snapshot. You can go back to any snapshot, compare versions, and see exactly what changed and when.

🤝

Collaboration Without Chaos

Multiple people can work on the same project at the same time. Git keeps track of who changed what and helps merge everything together.

🛡️

Safety Net

Experiment freely! Try a crazy refactor, rewrite a whole file — if it breaks, just go back to the last working commit. No fear.

6️⃣ Setting Up Git One-Time

Do this once on your computer, and you're ready to use Git forever.

First, check if Git is already installed:

bash — setup-git.sh
# Check git version (should show something like "git version 2.39.0") $ git --version git version 2.39.5 # Tell Git your name — this gets attached to every commit $ git config --global user.name "Your Name" # Tell Git your email (use the same one as your GitHub account) $ git config --global user.email "you@example.com"

These settings are stored once and used for every Git project on your computer. You only need to do this once.

💡 No Git? Download it free from git-scm.com or install via Homebrew: brew install git.

7️⃣ Your First Repository Let's Go

A repository ("repo") is a folder that Git is watching. Let's make one.

bash — first-repo.sh
# Create a project folder (if you don't have one already) $ mkdir ~/ai-projects $ cd ~/ai-projects # Create a Python file $ echo 'print("Hello, AI Engineer!")' > hello.py # Turn this folder into a Git repository $ git init Initialized empty Git repository in /Users/macbook/ai-projects/.git/ # THE most important command — check the status of your repo $ git status On branch main No commits yet Untracked files: (use "git add <file>..." to include in what will be committed) hello.py

git status is your best friend. Run it constantly. It tells you exactly what's going on: which files changed, what's staged, what branch you're on.

8️⃣ Making Your First Commit Snapshot Time

The two-step dance: git add + git commit.

bash — first-commit.sh
# Step 1: "Stage" your file — tell Git "I want to save this" $ git add hello.py # Check status — file is now "staged" (green in most terminals) $ git status # Step 2: Commit — take the snapshot with a message $ git commit -m "first commit: hello script" [main (root-commit) a1b2c3d] first commit: hello script 1 file changed, 1 insertion(+) # Check the log — see your snapshot history $ git log --oneline a1b2c3d (HEAD -> main) first commit: hello script
✏️ Edit your file → 📋 git add <file> → 💾 git commit -m "message" → 🔍 git status (check)
💡 Good commit messages: Write short, descriptive messages in present tense, like "add rate limit to API client" or "fix broken import path". Future you (and collaborators) will thank you.

9️⃣ GitHub — Your Code in the Cloud Backup & Share

Git is local on your computer. GitHub is where you back up and share your code.

What is GitHub?

GitHub is a website that hosts Git repositories in the cloud. It's like Dropbox for code — but way smarter. You can:

  • Back up your code so it's safe even if your laptop breaks
  • Share your code with others
  • Collaborate on projects with teammates
  • Show off your projects to employers

Push your code to GitHub

bash — push-to-github.sh
# Step 1: Create a new repo on github.com (no README, no .gitignore) # Step 2: Link your local repo to GitHub $ git remote add origin https://github.com/YOUR-USERNAME/ai-projects.git # Step 3: Push your commits to GitHub $ git push -u origin main Enumerating objects: 3, done. Counting objects: 100% (3/3), done. Writing objects: 100% (3/3), 247 bytes | 247.00 KiB/s, done. Total 3 (delta 0), reused 0 (delta 0), pack-reused 0 To https://github.com/YOUR-USERNAME/ai-projects.git * [new branch] main -> main Branch 'main' set up to track remote branch 'main' from 'origin'. # Now visit https://github.com/YOUR-USERNAME/ai-projects to see your code!
💡 Need a GitHub account? Go to github.com and sign up for free. Then create a new repository by clicking the green "New" button on your profile page.

🔟 Branching Basics Safe Experimentation

Branches let you work on new features without breaking your working code.

A branch is like a separate timeline. You start from your main timeline, make changes on a branch, and when everything works — you merge it back. Your main code stays safe the whole time.

bash — branching.sh
# List your branches (* means you're on it) $ git branch * main # Create and switch to a new branch in one command $ git checkout -b feature-greeting Switched to a new branch 'feature-greeting' # Make changes on this branch $ echo 'print("Hello from feature branch!")' > greeting.py $ git add greeting.py $ git commit -m "add greeting script" [feature-greeting b2c3d4e] add greeting script # Switch back to main $ git checkout main Switched to branch 'main' # Merge your feature branch into main $ git merge feature-greeting Updating a1b2c3d..b2c3d4e Fast-forward greeting.py | 1 + 1 file changed, 1 insertion(+) # See the history with your branch $ git log --oneline --graph * b2c3d4e (HEAD -> main, feature-greeting) add greeting script * a1b2c3d first commit: hello script
CommandWhat it does
git branchList all branches (the * shows which one you're on)
git checkout -b <name>Create a new branch AND switch to it
git checkout <name>Switch to an existing branch
git merge <branch>Merge changes from another branch into your current one
git log --oneline --graphSee commit history as a visual graph

1️⃣1️⃣ Essential CLI Tools Superpowers

curl fetches web data, jq reads JSON, and pipes (|) chain commands together.

curl — Make Web Requests

curl is like a web browser for the terminal. You give it a URL, and it fetches the content.

bash — curl-jq.sh
# Basic GET request — fetch a webpage $ curl https://api.github.com { "current_user_url": "https://api.github.com/user", "current_user_authorizations_html_url": "...", ... } # Add -s (silent) to hide progress output $ curl -s https://api.github.com/users/octocat

jq — Parse JSON

jq lets you extract specific data from JSON. The pipe symbol | sends output from one command into another.

bash — using-jq.sh
# The pipe (|) sends curl's output into jq # Extract just the "name" field $ curl -s https://api.github.com/users/octocat | jq .name "The Octocat" # Get the user's public repo count $ curl -s https://api.github.com/users/octocat | jq .public_repos 8 # Get the login and URL together $ curl -s https://api.github.com/users/octocat | jq '{login, html_url}' { "login": "octocat", "html_url": "https://github.com/octocat" }

Pipes — The Secret Sauce

The pipe | is the most powerful operator in the terminal. It takes the output of one command and feeds it as input to the next.

bash — pipes.sh
# Chain multiple commands together # ls → list files, then pipe into wc to count them $ ls | wc -l 12 # curl → jq → extract just what you want $ curl -s https://api.github.com/users/octocat | jq '.name, .bio' "The Octocat" "Some bio text here" # Save output to a file with > $ curl -s https://api.github.com/users/octocat | jq .name > username.txt

⚠️ Pitfalls — Avoid These Read First

These mistakes trip up every beginner. Know them before you start the exercises.

🔥 Forgetting git add Before git commit
You edit a file, type git commit -m "fix thing" — and nothing happens! You must git add first. Git only commits what you've staged. Always run git status before committing to see what's staged.
🚨 rm is Permanent — No Undo
On macOS, dragging a file to Trash lets you recover it. rm in the terminal does NOT use the Trash. Once deleted, it's gone. Always double-check the file name before hitting Enter.
🧨 Closing Terminal Kills Your Running Processes
If you start a Python script with python app.py and close the terminal window, the script dies. To keep it alive, prefix with nohup: nohup python app.py &.
💀 Forgetting -m on git commit Opens Vim
If you type git commit without -m "message", Git opens a text editor called Vim. To exit Vim: press Esc, type :q!, press Enter. Then use git commit -m "your message" next time.
🔑 Committing API Keys or Secrets
If you commit a file with an API key, that key is in your Git history forever — even if you delete it in a later commit. Anyone can read your history. Use a .env file for secrets and add .env to a .gitignore file so Git ignores it.
📏 Using git add . Without Checking What You're Staging
git add . stages everything in the folder — including compiled files, logs, large datasets, and secrets. Always run git status first to see what would be staged. Create a .gitignore file to tell Git which files to always skip.

Exercise 1

Your First Git Project

Create a project folder, make a Python file, initialize Git, and make your first commit.

  1. Open Terminal and create a folder called ~/my-first-project
  2. Change into that folder with cd
  3. Create a file called hello.py that prints "Hello, Git!"
  4. Run git init to turn the folder into a Git repository
  5. Run git status — see how hello.py shows as "untracked"?
  6. Run git add hello.py to stage it
  7. Run git status again — now it's green!
  8. Run git commit -m "first commit: hello world"
  9. Run git log --oneline to see your first commit
✅ Success Criteria
  • git log --oneline shows at least 1 commit with your message
  • git status shows "nothing to commit, working tree clean"
  • The hello.py file exists and prints "Hello, Git!"
Exercise 2

Branches and Merging

Create a feature branch, make changes, and merge them back into main.

  1. In the same ~/my-first-project folder, create a new branch: git checkout -b feature-greeting
  2. Create a new file greeting.py that prints "Hello from the feature branch!"
  3. Stage and commit the file with message feat: add greeting script
  4. Switch back to main: git checkout main
  5. Notice greeting.py disappears! (It only exists on the feature branch)
  6. Merge the feature branch: git merge feature-greeting
  7. Now greeting.py is back on main
  8. Run git log --oneline --graph to see the history
✅ Success Criteria
  • git log --oneline --graph shows at least 2 commits with branch info
  • Both hello.py and greeting.py exist on the main branch
  • Running python greeting.py prints "Hello from the feature branch!"
Exercise 3

GitHub + API Pipeline

Push your project to GitHub and use curl + jq to explore a public API.

  1. Go to github.com and create a new repository called my-first-project (don't add README or .gitignore)
  2. Copy the repository URL (looks like https://github.com/YOUR-USERNAME/my-first-project.git)
  3. In your terminal, run: git remote add origin PASTE_URL_HERE
  4. Push your code: git push -u origin main
  5. Visit your GitHub URL in a browser to verify the files are there
  6. Now use curl to fetch GitHub API info about yourself: curl -s https://api.github.com/users/YOUR-USERNAME
  7. Pipe the output into jq to extract your name and bio: curl -s https://api.github.com/users/YOUR-USERNAME | jq '{name, bio}'
  8. Save the output to a file: curl -s https://api.github.com/users/YOUR-USERNAME | jq '{name, bio}' > my-github-info.json
  9. Verify with cat my-github-info.json
✅ Success Criteria
  • Your GitHub repo shows your files in the browser
  • git log --oneline shows at least 2 commits (from exercises 1 & 2)
  • my-github-info.json exists with your name and bio from the API
  • The jq command successfully parsed the JSON and extracted specific fields

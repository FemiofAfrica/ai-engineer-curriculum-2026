Build an AI Agent

What it proves: You understand agentic planning, tool use, error recovery, and state management — where "prompting" ends and "engineering" begins.

Goal→ Plan→ Tool Call→ Observe→ Decide→ Next Action→ Complete

Project Spec

Minimum Viable Scope

  1. Build an agent that can use 3+ tools (e.g. web search, calculator, file read/write, API call)
  2. Accept a natural language goal and decompose it into steps
  3. Implement proper error handling — when a tool fails, the agent retries or adapts
  4. Surface the agent's reasoning (chain-of-thought or step-by-step plan visible to the user)
  5. Set max_steps = 8 and a hard timeout to prevent infinite loops

Agent Loop Core

agent.py
import json, time
from openai import OpenAI

client = OpenAI()

TOOLS = [
  {
    "type": "function",
    "function": {
      "name": "web_search",
      "description": "Search the web for current information",
      "parameters": {"type":"object","properties":{
        "query": {"type":"string"}
      },"required":["query"]}
    }
  },
  {
    "type": "function",
    "function": {
      "name": "run_code",
      "description": "Execute Python code (safe sandbox)",
      "parameters": {"type":"object","properties":{
        "code": {"type":"string"}
      },"required":["code"]}
    }
  },
  {
    "type": "function",
    "function": {
      "name": "read_file",
      "description": "Read a file from the workspace",
      "parameters": {"type":"object","properties":{
        "path": {"type":"string"}
      },"required":["path"]}
    }
  }
]

def run_tool(name, args):
  if name == "run_code":
    try:
      exec_globals = {}
      exec(args["code"], exec_globals)
      return {"result": "executed successfully"}
    except Exception as e:
      return {"error": str(e)}
  # web_search, read_file omitted for brevity
  return {"error": f"unknown tool: {name}"}

def agent_loop(goal, max_steps=8, timeout_sec=60):
  messages = [{"role":"user","content": goal}]
  start = time.time()

  for step in range(max_steps):
    if time.time() - start > timeout_sec:
      return "TIMEOUT: Agent exceeded time limit"

    resp = client.chat.completions.create(
      model="gpt-4o",
      messages=messages,
      tools=TOOLS
    )
    msg = resp.choices[0].message
    messages.append(msg)

    if not msg.tool_calls:
      return msg.content

    for tc in msg.tool_calls:
      args = json.loads(tc.function.arguments)
      for attempt in range(3):
        result = run_tool(tc.function.name, args)
        if "error" not in result:
          break
        time.sleep(2 ** attempt)
      messages.append({
        "role":"tool",
        "tool_call_id":tc.id,
        "content":json.dumps(result)
      })

  return f"MAX STEPS ({max_steps}) REACHED — partial result follows:\n{messages[-1].get('content','')}"

Stretch: Memory & State

Add conversation memory so the agent remembers past turns and can reference earlier results.

Stretch: Human-in-the-Loop

Pause before destructive actions (delete file, write to DB) and ask for confirmation.

Stretch: Multi-Agent

One orchestrator agent delegates subtasks to worker agents (researcher, writer, reviewer).

🧪 Deliverable

  1. Build an agent with ≥ 3 tools (web search, calculator, file operations, or your own API)
  2. Test it on a multi-step goal: "Research the latest AI agent frameworks, save a summary to research.md, and tell me the top 3 in order"
  3. Add a duplicate-call guardrail: if the agent calls the same tool with the same args twice, stop and return partial results
  4. Add a timeout mechanism (hard stop after 60 seconds regardless of max_steps)
  5. Push the project to a public GitHub repo with a README showing example runs and agent reasoning traces
  6. Write a thread showing: the agent's reasoning trace, a failure case (tool error), and how recovery worked
Success criteria: Agent completes multi-step goals in ≤ 8 steps. Error recovery is demonstrated (tool fails → agent adapts). Guardrail fires at least once. Repo is public with traces.

Module 7: Agent Frameworks & Tool Use

Build agents that plan, reason, call tools, and recover from failures — autonomously.

🎯 Before you start: This module assumes you've completed Modules 1–6. You should be comfortable building RAG pipelines, working with LLM APIs, and managing Python projects with git. If you haven't built the RAG project yet, do that first — this module makes heavy use of retrieval patterns.
Goal→ Plan→ Tool Call→ Observe→ Decide→ Next Action→ Complete

Core Concepts

Function / Tool Calling

LLM declares a structured tool call; you execute it and return the result. The model decides when and which tool to invoke.

Planning & Decomposition

Break a complex goal into steps. The agent reasons about what to do first, what depends on what, and how to sequence actions.

Error Recovery

Tools fail — API timeouts, invalid inputs, partial results. The agent must detect, retry, or adapt rather than collapse.

Memory & State

Multi-turn agents need conversation history and persistent state between steps — short-term (session) and long-term (vector store).

Guardrails & Safety

Limit tool scopes, set max steps, add human-in-the-loop for destructive actions, validate outputs before execution.

Multi-Agent Patterns

One orchestrator delegates to worker agents. Or specialised agents (researcher, writer, reviewer) collaborate on a single goal.

Minimal Tool-Calling Agent

agent.py
import json, httpx
from openai import OpenAI

client = OpenAI()

TOOLS = [
  {
    "type": "function",
    "function": {
      "name": "search_web",
      "description": "Search the web for current info",
      "parameters": {"type":"object","properties":{
        "query":{"type":"string","description":"search query"}
      },"required":["query"]}
    }
  },
  {
    "type": "function",
    "function": {
      "name": "calculate",
      "description": "Perform a calculation",
      "parameters": {"type":"object","properties":{
        "expr":{"type":"string","description":"math expression"}
      },"required":["expr"]}
    }
  }
]

def call_tool(name, args):
  if name == "calculate":
    return {"result": eval(args["expr"])}
  if name == "search_web":
    resp = httpx.get(f"https://api.duckduckgo.com/?q={args['query']}&format=json")
    return {"result": resp.json()["AbstractText"]}
  return {"error": "unknown tool"}

def run_agent(goal, max_steps=5):
  messages = [{"role":"user","content":goal}]
  for step in range(max_steps):
    resp = client.chat.completions.create(
      model="gpt-4o",
      messages=messages,
      tools=TOOLS
    )
    msg = resp.choices[0].message
    messages.append(msg)

    if not msg.tool_calls:
      return msg.content  # done

    for tc in msg.tool_calls:
      result = call_tool(
        tc.function.name,
        json.loads(tc.function.arguments)
      )
      messages.append({
        "role":"tool",
        "tool_call_id":tc.id,
        "content":json.dumps(result)
      })
  return "Max steps reached"

print(run_agent("What's 15% of 320? Then search the web for 'AI agents 2026'"))

Error Recovery Pattern

retry_agent.py
import time, random

def safe_call_tool(name, args, retries=3):
  for attempt in range(retries):
    try:
      if random.random() < 0.4:  # simulate failures
        raise ConnectionError("API timeout")
      return call_tool(name, args)
    except Exception as e:
      if attempt == retries - 1:
        return {"error": str(e), "status": "permanent_failure"}
      wait = 2 ** attempt  # exponential backoff
      time.sleep(wait)
  return {"error": "unreachable"}

⚠️ Pitfalls

Infinite loops. Always set max_steps and a timeout. Agents can and will call tools forever if you don't gate them.
No error recovery. Tools fail. If your agent crashes on the first API timeout, it's not production-ready. Always wrap tool calls in retry+adapt logic.
Uncontrolled tool permissions. Don't give an agent write access to your filesystem or database without human approval. Sanitise tool inputs.
No state management. Agents need memory between turns. If you don't store conversation history, they'll repeat themselves or lose context.
Over-reliance on frameworks. LangGraph/CrewAI are great, but understand raw tool calling first. Frameworks abstract away the mechanics you need to debug.

✨ What is Python?

A gentle introduction to what we're learning and why.

Python is a programming language — a way to give instructions to a computer. Think of it like a recipe: you write down a series of steps (the "code"), and the computer follows them one at a time. Python was designed to be easy to read and write, which is why it's the most popular language for beginners and for building AI systems.

In this course, you'll use Python to load data, call AI models, save results, and build tools. But first, you need to learn the basics — the alphabet before the sentences. By the end of this module, you'll write your own Python scripts from scratch.

🛠️ Getting Set Up

Let's make sure Python is installed on your computer.

Step 1: Check if Python is installed

Open your terminal (Terminal on Mac, Command Prompt on Windows). Then type:

terminal 
python3 --version

If you see something like Python 3.11 or Python 3.12, you're good! If you get an error, download Python from python.org/downloads.

Step 2: Open the Python REPL

A "REPL" is an interactive Python playground. Run this command:

terminal 
python3

You'll see something like >>>  — that means Python is waiting for you to type something.

💡 Pro tip You can exit the REPL by typing exit() and pressing Enter, or pressing Ctrl+D (Mac/Linux) / Ctrl+Z (Windows).

Step 3: Run a file

Create a new file called hello.py in a folder on your computer. Add this line:

hello.py 
print("Hello, world!")

Then run it from the terminal:

terminal 
python3 hello.py

You should see: Hello, world! printed on the screen. You just wrote and ran your first program!

📝 Your First Program

Printing, variables, and simple math — the building blocks.

print() — showing things on screen

print() is a command that tells Python to display something. Whatever you put inside the parentheses gets shown:

hello.py 
print("Hello, AI Engineering!")
print(42)
print("I am", 25, "years old")
output
Hello, AI Engineering!
42
I am 25 years old
🔤 Strings vs Numbers Text in quotes ("hello") is called a string. Numbers without quotes (42) are numbers. Python treats them differently — you can do math with numbers, but not with strings!

Variables — storing information

A variable is like a labeled box where you store data. You give it a name and a value with the = sign:

variables.py 
name = "Alice"
age = 25
is_student = True

print(name)
print(age)
print(f"Hi, my name is {name}!")
output
Alice
25
Hi, my name is Alice!

Notice the f before the string in the last line? That's called an f-string — it lets you plug variables directly into your text using curly braces {}. You'll use this all the time.

Simple math

Python can do arithmetic just like a calculator:

math.py 
a = 10
b = 3

print(a + b)    # addition → 13
print(a - b)    # subtraction → 7
print(a * b)    # multiplication → 30
print(a / b)    # division → 3.333...
print(a // b)   # integer division → 3
print(a ** b)   # exponent (10 to the power 3) → 1000
output
13
7
30
3.3333333333333335
3
1000

🧠 Making Decisions

Booleans, comparisons, and if/elif/else.

True or False

Python has a special type called Boolean — it's either True or False. You get them by comparing things:

booleans.py 
print(5 > 3)     # True — 5 is greater than 3
print(5 < 3)     # False — 5 is not less than 3
print(5 == 5)    # True — 5 equals 5 (note: double equals!)
print(5 != 3)    # True — 5 is not equal to 3
print(5 >= 5)    # True — 5 is greater than or equal to 5
output
True
False
True
True
True

if / elif / else — making choices

You can tell Python to do different things depending on conditions. The key rule: indentation matters. The code inside the if block must be indented (pressed Tab or 4 spaces).

conditions.py 
score = 85

if score >= 90:
    print("Grade: A")
elif score >= 80:
    print("Grade: B")     # ← this runs because 85 >= 80
else:
    print("Grade: C or lower")
output
Grade: B
⚠️ Indentation is not optional! In Python, spacing tells the computer which code belongs together. If your indentation is wrong, Python will give you an error. Always use 4 spaces (or 1 Tab) for each level of indentation.

📋 Lists and Loops

Working with multiple items at once.

What is a list?

A list is a collection of items, written with square brackets []. You can store anything in a list — numbers, strings, or a mix:

lists.py 
fruits = ["apple", "banana", "cherry"]
numbers = [1, 2, 3, 4, 5]
mixed = ["hello", 42, True]

print(fruits)        # ['apple', 'banana', 'cherry']
print(fruits[0])    # 'apple'  — lists start at index 0!
print(fruits[1])    # 'banana'
print(fruits[-1])   # 'cherry'  — -1 gets the last item
output
['apple', 'banana', 'cherry']
apple
banana
cherry
🔢 Indexing starts at 0 The first item in a list is at position 0, not 1. This is confusing at first, but you'll get used to it!

for loops — doing something with each item

A for loop lets you go through each item in a list, one at a time:

for_loop.py 
topics = ["AI", "Python", "Data"]

for topic in topics:
    print(f"Learning about {topic}...")
output
Learning about AI...
Learning about Python...
Learning about Data...

range() — counting loops

If you want to repeat something a specific number of times, use range():

range_demo.py 
for i in range(5):
    print(f"Count: {i}")
output
Count: 0
Count: 1
Count: 2
Count: 3
Count: 4

📦 Functions

Reusable blocks of code. No type hints, no decorators — just the basics.

A function is a reusable block of code. You define it once with def, then call it by name whenever you need it:

functions.py 
# Define a function
def greet(name):
    return f"Hello {name}!"

# Call the function
message = greet("Alice")
print(message)

# Functions can take multiple parameters
def add(x, y):
    return x + y

result = add(3, 7)
print(result)
output
Hello Alice!
10
💡 What is return? return sends a value back to wherever the function was called. If you don't use return, the function returns None (Python's way of saying "nothing").

📖 Dictionaries

Storing data as key-value pairs — the backbone of JSON and APIs.

A dictionary (or dict for short) stores data in key-value pairs. Think of it like a phonebook: you look up a name (the key) and get a number (the value):

dicts.py 
# Create a dictionary
user = {
    "name": "Alice",
    "score": 95,
    "active": True
}

# Access values by key
print(user["name"])   # Alice
print(user["score"])  # 95

# Add or update a value
user["email"] = "alice@example.com"
print(user)

# Loop through keys and values
for key in user:
    print(f"{key}: {user[key]}")
output
Alice
95
{'name': 'Alice', 'score': 95, 'active': True, 'email': 'alice@example.com'}
name: Alice
score: 95
active: True
email: alice@example.com
🤖 Why dictionaries matter for AI When you call an AI API like OpenAI or Anthropic, the response comes back as a dictionary (or a list of dictionaries). Learning to work with dicts now means you'll be ready to handle real API responses later.

📂 Working with Files

Reading and writing text and JSON files.

Python can read and write files on your computer. The with open() pattern is the standard way:

write_file.py 
data = ["AI", "Engineering", "Python"]

# Write to a file
with open("topics.txt", "w") as f:
    for topic in data:
        f.write(topic + "\n")

# Read the file back
with open("topics.txt", "r") as f:
    content = f.read()

print(content)
output
AI
Engineering
Python
💡 What does with do? The with statement makes sure the file is properly closed after we're done reading or writing — even if something goes wrong. You don't need to remember to call f.close(). Just use with open(...) as f: every time.

Working with JSON

JSON is the standard format for sending data between systems. Python's json module makes it easy:

json_demo.py 
import json

person = {
    "name": "Bob",
    "age": 30,
    "skills": ["Python", "AI"]
}

# Write to a JSON file
with open("person.json", "w") as f:
    json.dump(person, f, indent=2)

# Read the JSON file back
with open("person.json", "r") as f:
    loaded = json.load(f)

print(loaded["name"])
print(loaded["skills"])
output
Bob
['Python', 'AI']

🚨 Handling Errors

When things go wrong, catch them gracefully.

Programs crash. That's normal. Python gives you try/except to handle errors without stopping your whole program:

errors.py 
try:
    with open("nonexistent.txt", "r") as f:
        content = f.read()
        print(content)
except FileNotFoundError:
    print("That file doesn't exist! But the program kept going.")

print("This line runs no matter what.")
output
That file doesn't exist! But the program kept going.
This line runs no matter what.

Common beginner errors

common_errors.py 
# NameError: misspelled a variable name
name = "Alice"
print(nme)            # NameError: name 'nme' is not defined

# TypeError: mixed up types
print("Age: " + 25)   # TypeError: can only concatenate str (not "int") to str

# IndentationError: spacing is wrong
#   print("hello")   # ← this line has wrong indentation
print("ok")          # ← this one is fine

📚 Using Libraries

Installing and importing code that other people wrote.

Python has a huge ecosystem of libraries — code packages written by other people that you can use. You install them with pip:

terminal 
pip install requests

Then you can use it in your code:

library_demo.py 
import requests

response = requests.get("https://api.github.com")
data = response.json()

print(data["current_user_url"])
output
https://api.github.com/user
📦 What is import? import loads a library into your program so you can use its functions. Think of it like borrowing a toolbox — someone else built the tools, you just use them.

🧪 Virtual Environments

Keeping each project's dependencies separate.

A virtual environment is an isolated space for each Python project. It keeps libraries from one project from interfering with another. Here's the workflow:

terminal 
# Step 1: Create a virtual environment
python3 -m venv myenv

# Step 2: Activate it
# On Mac/Linux:
source myenv/bin/activate

# On Windows:
# myenv\Scripts\activate

# Step 3: Install libraries inside the environment
pip install requests

You'll know it's active because you'll see (myenv) at the start of your terminal prompt. To leave the environment, type deactivate.

⚠️ Always use a virtual environment! Never install project libraries globally. Each project gets its own virtual environment. This is one of the first things professionals check.

🎯 Bringing It All Together

A short script that uses everything you've learned.

Here's a complete script that creates a list of AI topics, loops through them, stores the data in a dictionary, saves it to a JSON file, and reads it back — all with error handling:

final_script.py 
import json

def process_topics(topic_list):
    """Turns a list of topics into score data."""
    results = []
    for topic in topic_list:
        entry = {
            "topic": topic,
            "length": len(topic),
            "processed": True
        }
        results.append(entry)
    return results

def save_to_json(data, filename):
    with open(filename, "w") as f:
        json.dump(data, f, indent=2)
    print(f"Saved to {filename}")

def load_from_json(filename):
    try:
        with open(filename, "r") as f:
            return json.load(f)
    except FileNotFoundError:
        print("File not found!")
        return []

# --- Run it! ---
topics = ["RAG", "Agents", "Fine-tuning", "Embeddings"]
results = process_topics(topics)
save_to_json(results, "ai_topics.json")
loaded = load_from_json("ai_topics.json")

for item in loaded:
    print(f"{item['topic']}: {item['length']} characters")
output
Saved to ai_topics.json
RAG: 3 characters
Agents: 6 characters
Fine-tuning: 11 characters
Embeddings: 10 characters

🏗️ How Python Works

A simple picture of how code becomes output.

You write code → Python interprets it → You get output. Variables, functions, files, and libraries are the four tools you'll use most.

⚠️ Common Pitfalls

Mistakes every beginner makes — learn to spot them early.

  • 🔥
    Forgetting the colon — Every if, for, def, and class line must end with a colon (:). if score > 50: ✅ — if score > 50 ❌
  • 🚨
    Indentation errors — Python uses indentation (spaces/tabs) to know which code belongs inside a block. Mixed tabs and spaces or missing indentation = error. Always use 4 spaces.
  • 🧨
    Forgetting to activate your virtual environment — You install a library, your code says ModuleNotFoundError. You probably installed it globally but your project's venv isn't active. source myenv/bin/activate first!
  • 💣
    Variable name errors (typos) — name = "Alice" then print(nme) gives a NameError. Python is case-sensitive — Name and name are different!
  • 🔑
    Hardcoding secrets — Never put API keys, passwords, or tokens directly in your code. Use environment variables or a .env file. And never commit secrets to GitHub!
  • 📏
    Confusing == and = — = assigns a value (x = 5). == checks if two things are equal (if x == 5). Using the wrong one breaks your code.

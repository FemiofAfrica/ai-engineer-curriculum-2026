Prompt Engineering for Code Assistants

prompt_patterns.md
# You get better code when you give better context.

## Good prompts
"Add a retry decorator with exponential backoff for async functions"
"Refactor this into a class with type hints, dataclasses, and a factory method"
"What are the edge cases in this rate limiter? Write tests for each."

## Bad prompts
"Write code" — too vague
"Make it better" — what does better mean?
"Fix the bug" — where? what bug?

## The "Explain" Pattern (from @cyrilXBT)
After generating code, say:
"Walk me through this line by line, explain alternatives,
and identify what will break in production."

This forces you to understand every line.
The AI can hallucinate — you are the reviewer.

Workflow: Ship with Claude Code

terminal: claude cli
# Inside your project
claude

# Then in the Claude Code conversation:
# 1. "Read the README and explain the architecture"
# 2. "Add a /health endpoint that returns DB latency and API key status"
# 3. "Write tests for the /ask endpoint — edge cases: empty query, rate limit, missing API key"
# 4. "Show me the diff of what you changed"
# 5. "Create a PR description summarizing these changes"

# Claude Code edits files, runs commands, commits, and opens PRs.
# YOU review every diff before approving.

⚠️ Pitfalls

You are the reviewer, not the passenger. AI writes code you don't understand = tech debt you can't fix. Always read and understand every diff before approving.
Hallucinated imports and APIs. AI models invent function signatures, package names, and endpoints. Verify against docs before running.
Context management. Tools lose context after long conversations. Break big tasks into focused sessions. Use @-mentions and file references to re-establish context.
Over-reliance. If you can't write the code yourself, you can't review it either. Use AI to go faster, not to skip learning.
Security blind spots. AI doesn't check for SQL injection, hardcoded secrets, or permission issues. Add a security review step to your workflow.

🧪 Exercise: Ship Something With AI Tools

Use Claude Code or Cursor to build and ship a small AI feature — start to finish.

  1. Use Claude Code (or Cursor) to build a simple CLI tool: "a tool that takes a URL, fetches the page text, summarises it with an LLM, and saves the summary to a markdown file."
  2. For every code block the AI generates, ask it to "walk me through this line by line." Paste the explanation into a NOTES.md file in the project.
  3. Ask the AI to add tests and error handling.
  4. Review the diff. Find at least one thing the AI got wrong (hallucinated import, wrong error handling, missing edge case). Fix it manually.
  5. Commit and push the project.
Success criteria: The CLI runs end-to-end. NOTES.md has your line-by-line walkthrough. You found and fixed at least one AI mistake. The repo is public on GitHub.
